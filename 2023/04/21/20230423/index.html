简单记录重新折腾的经历

Author: Yonkyin Date: April 21, 2023  16:00 Category: Computer Science

无聊了，想试着重新开始写写博客。

是的，重新开始。

这次还是打算选用 Hexo + Github Page 来部署，不用弄云服务器，简单弄个域名弄个解析就行了，就是每次写东西后部署很烦。希望后期积累多了后，它不会变卡。


环境搭建


  1. 安装 git ；
  2. 安装 Node.js ；
  3. 安装 Hexo ；
  4. 创建 Github 仓库。

安装环境的步骤比较简单，git 和 Node.js 都可以直接官网下载安装包，简单记录一下其中的步骤。

连接 Github：

右键 -> Git Bash Here，设置用户名和邮箱：

创建 SSH 密匙：

输入 ssh-keygen -t rsa -C "GitHub 邮箱" ，然后一路回车。

添加密匙：

进入 [C:\Users\用户名\.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

验证连接：

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。

显示 “Hi xxx! You’ve successfully……” 即连接成功。

新建一个文件夹用来存放 Hexo 的程序文件，打开该文件夹，右键 终端 或 Git Bash Here。（Windows端）

这时访问 GitHub 域名 https://<用户名>.github.io 就可以看到 Hexo 网站了。

设置域名跳转：

先通过阿里云等，设置 DNS 解析，将自己注册的域名解析到 <用户名>.github.io ，然后登陆 Github 设置自定义域名。

登录 Github ，进入博客的 repo $\to$ Settings $\to$ Pages $\to$ Custom domain ，然后点 Save 保存。

完成！
